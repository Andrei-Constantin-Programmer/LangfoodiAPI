Architectural Decision Records

This is the folder containing all ADRs for the API.

What are ADRs?

Architectural Decision Records are documents written in markdown (.md) that chronicle important architectural decisions for future reference.

More information about ADRs can be found on the Confluence wiki (MGT.4.8)

ADRs

2023-05-25 Adoption of the Mediator pattern Using the mediator pattern (e.g. through MediatR) to publish requests from endpoints to the Application Layer

2023-09-29 Adoption of Contracts for defining client-to-API interactions Use Contract DTOs to clean up incoming requests and to allow for specialised incoming DTOs
2024-03-06 Renaming the "Core" and "DataAccess" modules to "Presentation" and "Infrastructure" Align layer names to the Clean Architecture and industry standards, and remove confusion between the Presentation and Domain layers
2024-03-07 Authorising the API with JWTs Use JSON Web Tokens to authorise incoming requests to disallow bad actors from interacting with the system (with some exceptions, like registering)
